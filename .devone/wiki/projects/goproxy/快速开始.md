# 快速开始

这个仓库默认提供两条启动路径：Docker Compose 是推荐入口；本地 Go 运行适合开发和调试。若要使用订阅中的加密节点，本地环境还需要可执行的 `sing-box`。

## 前置条件

### Docker 路径

- 已安装 Docker / Docker Compose
- 需要暴露端口：
  - `7776`：HTTP 最低延迟
  - `7777`：HTTP 随机轮换
  - `7778`：WebUI
  - `7779`：SOCKS5 随机轮换
  - `7780`：SOCKS5 最低延迟

### 本地 Go 路径

- Go `1.25.x`
- `CGO_ENABLED=1`（因为依赖 `github.com/mattn/go-sqlite3`）
- 若启用订阅中的加密节点：系统中应能找到 `sing-box`，或通过 `SINGBOX_PATH` 指定其路径

## Docker Compose 启动

```bash
cp .env.example .env
docker compose up -d
```

启动后默认访问：

- WebUI：`http://localhost:7778`
- 默认密码：`goproxy`

### 关键环境变量

```env
WEBUI_PASSWORD=your_password
PROXY_AUTH_ENABLED=true
PROXY_AUTH_USERNAME=proxy
PROXY_AUTH_PASSWORD=secret
BLOCKED_COUNTRIES=CN
ALLOWED_COUNTRIES=
CUSTOM_PROXY_MODE=mixed
DATA_DIR=/app/data
SINGBOX_PATH=sing-box
```

## 本地运行

```bash
go run .
```

或先编译再运行：

```bash
go build -o proxygo .
./proxygo
```

若只想调试免费代理功能，可不安装 `sing-box`；但一旦使用订阅中的加密节点，`custom/singbox.go` 会在刷新时调用它。

## 最小验证路径

### HTTP 代理

```bash
curl -x http://localhost:7777 https://httpbin.org/ip
curl -x http://localhost:7776 https://httpbin.org/ip
```

### SOCKS5 代理

```bash
curl --socks5 localhost:7779 https://httpbin.org/ip
curl --socks5 localhost:7780 https://httpbin.org/ip
```

### 使用仓库自带脚本

```bash
./test/test_proxy.sh 7777
./test/test_proxy.sh 7776
./test/test_socks5.sh 7779
./test/test_socks5.sh 7780
```

## 数据与配置位置

- 容器部署：默认挂载卷 `goproxy-data:/app/data`
- 本地运行：若未设置 `DATA_DIR`，`config.json` 与 `proxy.db` 写在当前工作目录
- 通过 WebUI 保存配置后，会生成 `config.json`
