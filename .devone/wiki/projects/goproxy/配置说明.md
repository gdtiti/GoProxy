# 配置说明

GoProxy 的配置分两层：环境变量决定首次启动和进程外注入；`config.json` 负责持久化 WebUI 可修改项。当前实现中，`config.json` 对可持久化字段的优先级高于环境变量。

## 配置来源与优先级

### 环境变量

主要由 `DefaultConfig()` 读取：

- `WEBUI_PASSWORD`
- `PROXY_AUTH_ENABLED`
- `PROXY_AUTH_USERNAME`
- `PROXY_AUTH_PASSWORD`
- `BLOCKED_COUNTRIES`
- `ALLOWED_COUNTRIES`
- `CUSTOM_PROXY_MODE`
- `SINGBOX_PATH`
- `DATA_DIR`

### `config.json`

- 路径：`ConfigFile()` 返回 `DATA_DIR/config.json`；若未设置 `DATA_DIR`，则位于当前工作目录
- 由 `config.Save()` 写入
- 会覆盖对应环境变量派生出的默认值，尤其是池子参数、延迟阈值、健康检查、优化、地理过滤和订阅参数

## 核心配置项

### 服务端口

- `WebUIPort`：默认 `:7778`
- `ProxyPort`：默认 `:7777`
- `StableProxyPort`：默认 `:7776`
- `SOCKS5Port`：默认 `:7779`
- `StableSOCKS5Port`：默认 `:7780`

### 代理认证

- `PROXY_AUTH_ENABLED=true` 开启代理认证
- HTTP 代理使用 `Proxy-Authorization: Basic ...`
- SOCKS5 代理在握手阶段做用户名 / 密码认证
- WebUI 自己使用独立的 session 登录，不复用代理认证

### 地理过滤

- 默认黑名单：`CN`
- 若 `ALLOWED_COUNTRIES` 非空，则白名单优先，黑名单被忽略
- 该规则同时影响：
  - 免费代理入池
  - 启动时历史数据清理
  - 订阅代理启用 / 恢复

### 池子与延迟

- `PoolMaxSize`
- `PoolHTTPRatio`
- `PoolMinPerProtocol`
- `MaxLatencyMs`
- `MaxLatencyEmergency`
- `MaxLatencyHealthy`
- `MaxLatencyDegradation`
- `ReplaceThreshold`

### 验证与后台任务

- `ValidateConcurrency`
- `ValidateTimeout`
- `ValidateURL`
- `HealthCheckInterval`
- `HealthCheckBatchSize`
- `OptimizeInterval`
- `IPQueryRateLimit`

### 订阅相关

- `CustomProxyMode`：`mixed` / `custom_only` / `free_only`
- `CustomPriority` / `CustomFreePriority`：混用模式优先级
- `CustomProbeInterval`
- `CustomRefreshInterval`
- `SingBoxPath`
- `SingBoxBasePort`

## 数据目录

### `DATA_DIR` 已设置时

- `proxy.db`
- `config.json`
- `subscriptions/*.yaml`
- `sing-box` 运行配置及相关文件

### `DATA_DIR` 未设置时

- `config.json` 与 `proxy.db` 落到当前目录
- 访客或管理员上传的订阅文件也会以相对路径落盘

## 运行时变更路径

1. WebUI `POST /api/config/save`
2. `config.Save(&newCfg)`
3. 写入 `config.json`
4. 更新全局内存配置
5. 通过 channel 通知主流程做池容量调整

## 已确认 / 推断 / 未知

### 已确认

- `config.Get()` / `config.Save()` 通过读写锁保护全局配置。
- `savedConfig` 只持久化部分字段；例如端口和数据库路径不在 WebUI 可修改持久化范围内。

### 推断

- 当前配置设计默认把“部署层参数”与“运行时池参数”分层：前者主要靠环境变量，后者主要靠 WebUI + `config.json`。

### 未知

- 仓库内没有单独的配置 schema 文档；字段变更时主要依赖代码与 WebUI 前端同步维护。
