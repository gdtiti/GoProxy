# 仓库地图

这个仓库的代码入口集中在根目录与 10 个一级模块目录，没有 `cmd/`、`internal/` 之类更深层的拆分；阅读时先看 `main.go`，再按“配置 → 存储 → 抓取 / 验证 / 池化 → 订阅 → 对外服务 → WebUI”顺序推进最省时间。

```text
GoProxy/
├─ main.go

```

## 目录说明

### `config/`

- 用途：全局配置加载、保存、槽位与阈值计算。
- 关键文件：`config/config.go`
- 是否核心：是

### `storage/`

- 用途：SQLite schema、迁移、代理 / 订阅 / 源状态 CRUD。
- 关键文件：`storage/storage.go`
- 是否核心：是

### `fetcher/`

- 用途：免费代理抓取、源选择、断路器。
- 关键文件：`fetcher/fetcher.go`、`fetcher/source_manager.go`
- 是否核心：是

### `validator/`

- 用途：连通性、延迟、出口 IP 与地理位置验证。
- 关键文件：`validator/validator.go`
- 是否核心：是

### `pool/`

- 用途：容量、协议比例、替换策略、池状态。
- 关键文件：`pool/manager.go`
- 是否核心：是

### `checker/`

- 用途：后台健康检查。
- 关键文件：
  - `checker/health_checker.go`：主流程实际使用
  - `checker/checker.go`：仓库中保留的旧实现风格
- 是否核心：是，但内部存在新旧两套实现痕迹

### `optimizer/`

- 用途：健康状态下替换慢代理。
- 关键文件：`optimizer/optimizer.go`
- 是否核心：中

### `custom/`

- 用途：订阅拉取、解析、转换、验证、刷新、探测唤醒。
- 关键文件：`custom/manager.go`、`custom/parser.go`、`custom/singbox.go`
- 是否核心：是

### `proxy/`

- 用途：对外 HTTP / SOCKS5 服务。
- 关键文件：`proxy/server.go`、`proxy/socks5_server.go`
- 是否核心：是

### `webui/`

- 用途：页面与 API。
- 关键文件：`webui/server.go`、`webui/html.go`
- 是否核心：是

### `logger/`

- 用途：内存日志供 WebUI 查看。
- 关键文件：`logger/logger.go`
- 是否核心：中

### `test/`

- 用途：手工验证脚本，不是 Go 单元测试目录。
- 关键文件：`test/README.md`、`test_proxy.sh`、`test_socks5.sh`、`test_proxy.py`、`test_proxy.go`
- 是否核心：中

## 关键入口

- 应用入口：`main.go`

- 存储入口：`storage/storage.go`
- WebUI 入口：`webui/server.go`

## 已确认 / 推断 / 未知

### 已确认

- 仓库中没有 `*_test.go` 文件，自动化测试并未采用 Go 原生 test 组织方式。
- `webui/html.go` 说明前端资源内嵌在 Go 代码中，而不是独立前端工程。

### 推断

- `POOL_DESIGN.md`、`DATA_DIRECTORY.md`、`GEO_FILTER.md` 是维护者用于解释当前实现的辅助设计文档，可信度高于普通 README 附录，但仍以源码为准。

### 未知

- 根目录 `proxygo` 是否应该纳入版本控制，仓库内没有说明。
