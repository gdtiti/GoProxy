# 首页

GoProxy 是一个单进程 Go 代理池服务：它抓取免费代理、导入订阅节点、把可用节点写入 SQLite，并同时暴露 4 个代理端口和 1 个 WebUI 端口。

## 项目用途

- 自动抓取公开 HTTP / SOCKS5 代理并做出口 IP、地理位置、延迟验证。
- 支持订阅 URL / 配置文件导入；加密节点通过 `sing-box` 转成本地 SOCKS5 节点。
- 提供两种 HTTP 模式、两种 SOCKS5 模式和一个 WebUI 管理面板。

## 技术栈

- 语言：Go `1.25.0`
- 主要依赖：`github.com/mattn/go-sqlite3`、`modernc.org/sqlite`、`golang.org/x/net/proxy`、`gopkg.in/yaml.v3`
- 持久化：SQLite（`proxy.db` + `config.json`）
- 运行辅助：可选 `sing-box` 子进程、Docker / Docker Compose

## 已确认 / 推断 / 未知

### 已确认

- `main.go` 在启动时顺序装配 `logger`、`config`、`storage`、`fetcher`、`validator`、`pool`、`checker`、`optimizer`、`custom`、`webui` 和代理服务。
- WebUI 监听 `:7778`，HTTP 代理监听 `:7777` / `:7776`，SOCKS5 代理监听 `:7779` / `:7780`。
- 正式持久化仅依赖 SQLite；仓库中未发现 Redis、MQ 或独立缓存层。
- `storage` 已支持按 CGO 可用性切换 SQLite driver：有 CGO 时走 `github.com/mattn/go-sqlite3`，无 CGO 时走 `modernc.org/sqlite`。

### 推断

- 当前项目更偏向“单体守护进程 + 嵌入式管理面板”，而不是多服务拆分部署。
