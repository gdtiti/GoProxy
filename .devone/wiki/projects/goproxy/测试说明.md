# 测试说明

这个仓库的测试资产以手工脚本为主，而不是 Go 原生 `*_test.go` 单元测试。当前更适合把它理解为“启动后验证代理行为”的操作手册集合。

## 当前测试形态

- Go 原生测试：未发现 `*_test.go`
- 手工脚本目录：`test/`
- 覆盖方向：
  - HTTP 随机 / 最低延迟端口
  - SOCKS5 随机 / 最低延迟端口
  - 持续请求下的出口 IP 轮换、延迟和丢包率观察

## 已发现测试文件

| 文件 | 作用 | 类型 |
| --- | --- | --- |
| `test/README.md` | 手工测试说明与命令导航 | 文档 |
| `test/test_proxy.sh` | HTTP 代理持续测试 | Bash |

## 推荐验证命令

### HTTP

```bash
./test/test_proxy.sh 7777
./test/test_proxy.sh 7776
```

### SOCKS5

```bash
./test/test_socks5.sh 7779
./test/test_socks5.sh 7780
```

### Go 版脚本

```bash
go run test/test_proxy.go
```

## 测试关注点

- `7777` / `7779`：出口 IP 是否轮换
- `7776` / `7780`：出口 IP 是否趋于稳定且延迟较低
- 丢包率是否保持在可接受范围
- 代理失败后，节点是否会被删除或禁用

## 当前缺口

- 没有自动化单元测试、属性测试或集成测试框架
- 仓库内没有 CI 配置来自动跑测试脚本
- `test/README.md` 提到 `TEST_GUIDE.md`，但当前树中未发现该文件

## 已确认 / 推断 / 未知

### 已确认

- 测试目录更接近运行后手工验收，而不是开发期回归测试。
- `test/test_proxy.go` 默认测试 `127.0.0.1:7777`，并持续访问 `ip-api.com`。

### 推断

- 当前维护策略更依赖 WebUI 观测和手工压测，而不是代码提交时的自动回归。

### 未知

- 这些测试脚本是否被外部发布流程复用，仓库内没有证据。
