## 实际改动摘要

- 本轮改动:
  - 新增 `webui/theme.go`，集中承载主题 token、主题选择器样式与前端主题切换脚本。
  - 修改 `webui/html.go`，登录页接入统一主题变量与主题选择器。
  - 修改 `webui/dashboard.go`，仪表盘接入统一主题变量与主题选择器，并保持原有 guest/admin 分支逻辑。
  - 新增 `storage/sqlite_driver_cgo.go` 与 `storage/sqlite_driver_nocgo.go`，按 CGO 可用性自动选择 SQLite driver。
  - 修改 `storage/storage.go`，把 driver 名切换为常量选择，消除对单一 `sqlite3` driver 的硬编码。
  - 修改 `go.mod` / `go.sum`，纳入 `modernc.org/sqlite` 及其依赖。
  - 修改 `README.md` / `CLAUDE.md`，同步本地运行不再强制要求 CGO。
- 涉及文件/模块:
  - `go.mod`
  - `go.sum`
  - `README.md`
  - `CLAUDE.md`
  - `webui/theme.go`
  - `webui/html.go`
  - `webui/dashboard.go`
  - `storage/storage.go`
  - `storage/sqlite_driver_cgo.go`
  - `storage/sqlite_driver_nocgo.go`
  - `.devone-runtime-data/theme_harness.py`（临时验证脚本，非正式仓库代码）
- 主题方案:
  - `graphite-dark`
  - `aurora-dark`
  - `mist-light`
  - `sand-light`
- 持久化策略:
  - 浏览器 `localStorage['goproxy.theme']`
  - 无保存值时按系统偏好回落到 `mist-light` 或 `graphite-dark`
  - 非法主题值显式回退到 `graphite-dark`
- 未纳入仓库提交的临时验证产物:
  - `.devone-runtime-data/theme_harness.py`
  - `.devone-runtime-data/login-mist-light.png`
  - `.devone-runtime-data/dashboard-admin-sand-light.png`

  - 页面级运行态验证借助临时 harness 完成，只用于绕过当前环境的 SQLite/CGO 启动阻塞，不伪装为真实 GoProxy 全栈后端。
