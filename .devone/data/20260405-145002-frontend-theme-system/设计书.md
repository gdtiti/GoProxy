## 方案摘要

- 推荐方案:
  - 在现有嵌入式前端结构上增加“语义化主题 token + JS 主题管理器 + `localStorage` 持久化”，统一接管登录页与仪表盘的视觉变量。
- 推荐原因:
  - 改动集中在 `webui/` 内部，不需要引入独立前端工程。
  - 与现有 `localStorage.lang` 的浏览器侧状态管理方式一致，接入成本低。
  - 可以先把“主题系统”抽出来，再逐步给所有组件接上 token，避免继续复制硬编码颜色。

## 备选方案对比

### 方案 A

- 做法:
  - 在 `webui/dashboard.go` 和 `webui/html.go` 中引入统一主题变量层。
  - 使用一组语义 token（背景、卡片、边框、文本、次文本、强调色、成功/警告/危险色、阴影等）驱动所有界面。
  - 用前端 JS 主题管理器负责：默认主题判定、主题切换、应用 token、写入/恢复 `localStorage`。
- 收益:
  - 保持当前项目结构不变，最小代价建立长期可维护的主题基础设施。
  - 登录页与仪表盘都能统一接入。
  - 后续新增主题只需补 token，不必重复改整页 CSS。
- 代价:
  - 需要梳理现有大量内联/硬编码颜色并替换为语义变量。
- 风险:
  - 若 token 设计不完整，后续组件仍会回到局部硬编码。

### 方案 B

- 做法:
  - 保持当前结构不变，仅为每个页面追加一套浅色和深色 CSS，并手工写切换逻辑。
- 收益:
  - 初期落地快。
- 代价:
  - 登录页与仪表盘会继续分裂维护。
  - 每多一套主题，重复 CSS 成本线性上涨。
- 风险:
  - 很快演变成多份大块样式复制，后续难维护，也难验证“便于阅读、护眼”是否一致成立。

## 锁定设计

- 入口模块:
  - `webui/dashboard.go`：主界面 `dashboardHTML`
  - `webui/html.go`：登录页 `loginHTML` / `loginHTMLWithError`
- 关键调用链:
  - `webui/server.go:79-80,143,146-167` 负责输出登录页与仪表盘 HTML
  - 前端启动阶段恢复 `localStorage.lang`；本轮将按同样模式增加 `localStorage.theme`
  - 页面初始化时应用主题 -> 再执行现有 i18n / 数据加载逻辑
- 数据流:
  - 主题预设定义在前端脚本中
  - 用户点击主题切换器 -> 更新当前主题 key -> 应用 CSS 变量 / `data-theme` -> 写入 `localStorage`
  - 页面加载时优先恢复保存的主题；若不存在，则使用默认主题（必要时参考系统 `prefers-color-scheme`）
- 错误处理:
  - 若读不到或识别不到主题 key，回退到默认主题，不影响页面渲染
  - 只允许显式回退到默认主题，不引入隐式多轨样式
- 配置变更:
  - 本轮不新增后端配置项，主题选择只保存在浏览器本地
  - 不改动 `config.json`、`apiConfigSave()` 与数据库

## 分阶段计划

### 阶段 1

- 目标:
  - 建立统一主题基础设施，完成 token 分层、主题预设结构和应用逻辑。
- 风险主题:
  - 现有 dashboard 与 login 两套样式结构不一致，公共抽象容易遗漏。
- 主要变更:
  - 梳理现有色板与阴影
  - 定义语义 token
  - 设计至少 4 套主题预设
  - 实现主题切换与持久化逻辑
- 验证方式:
  - 代码级检查：登录页与仪表盘均接入统一主题 key / token 体系
  - 页面级检查：切换后刷新仍保留主题
- 完成标志:
  - 不再依赖当前单一荧光绿配色作为唯一主题

### 阶段 2

- 目标:
  - 把主题系统铺到核心组件并做可读性收敛。
- 风险主题:
  - 表格、弹窗、按钮、状态色、hover 态等细节容易出现视觉断层。
- 主要变更:
  - 接入顶部导航主题切换入口
  - 调整表格、侧栏、弹窗、按钮、输入框、状态标签的 token 使用
  - 校正浅色/深色主题的文本对比、边框强度和护眼亮度
- 验证方式:
  - 静态代码核对 + 页面实际操作检查
  - 覆盖 guest/admin、登录页/仪表盘、语言切换与主题切换组合场景
- 完成标志:
  - 用户能稳定在多套浅/深主题间切换，主要界面无明显读写障碍

- 知识事实源:

  - `.devone/wiki/projects/goproxy/首页.md`
- 实现事实源:
  - `webui/html.go`
  - `webui/dashboard.go`
  - `webui/server.go`
